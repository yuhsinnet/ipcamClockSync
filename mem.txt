語言 c#
目標功能 
透過ONVIF 掃描攝影機 更新攝影機時間 提供單次手動更新或設定到目標ntp server 
把掃描到的攝影機ip 及連線帳密 統一存成無加密 json檔<下稱 攝影機清單>
做一個小型ntp server 供攝影機校時

提供 有二種啟動模式
1.使用 指令靜默操作 
1.1提供 /ntpserver start stop restart 啟動ntp server 可以的話寫成服務模式 <討論是否ntp server 是否獨立程式 一起佈署>
1.2提供 /a 使用清單中的攝影機連線資訊 透過單次時間更新攝影機時間 
1.3 <你幫我想想與補充>


2. 使用 console gui 的設定方式
2.1首先 軟體聲明<可選擇下次是否顯示 預設 下次不顯示> 後接 簡易說明<可選擇下次是否顯示 預設 下次不顯示> 
2.2 功能一 掃描攝影機
2.2.1 選擇後立刻用使用 onvif 的掃描協議進行掃描 時間大概 5 秒 期間有某種簡易動畫指示掃描中
2.2.2 掃描完畢後顯示一個有掃描到的清單 請注意 此請單有可能會過長 注意分頁問題 
2.2.3 清單下方 提問使用者 是否進階掃描 if 是 詢問登入帳密<一般來說所有攝影機會一致 問一次就好> 可以正常登入並取得時間的保留 if 否 進入下一個畫面(選擇要保存的攝影機)
2.2.4 選擇要保存的攝影機清單 在這階段可以個別設定攝影機的 帳密 保存包含 ip 帳號 密碼
2.2.5 詢問清單保存位置 預設應用程式目錄
2.2.6 詢問是否現在更新攝影機清單中的攝影機時間 說明 這是使用單次時間更新

2.3 功能二 自動單次更新 攝影機清單中的時間 <注意 更新時間時 每次的更新再讀一次系統時間 避免數量太多 導致的秒差>
2.3.1 詢問清單位置 預設應用程式目錄

2.4 功能三 自動設定 攝影機清單中的攝影機設定 ntp時間更新
2.4.1 詢問清單位置 預設應用程式目錄
2.4.1 詢問目標ntp server ip 或是 設定成本機 ip
2.5 功能四 設定
2.5.1 設定掃描時長
2.5.2 匯出設定檔、攝影機到指定目錄
2.5.3 使用 yaml 作設定檔
2.5.4 設定單次時間更新要不要用 多執行緒 一次機個執行緒非同步進行校時 單執行緒
2.5.5 設定攝影機無法連現時長 超過該秒數判定無法連線
2.5.6 <你幫我想想與補充>

單次時間更新:用onvif 對攝影機 時間設定更新方式手動 時區判讀本機時區推送給攝影機 推送本機時間
ntp時間更新:用onvif 對攝影機推送 目標ntp 主機 ip 

提供 兩種佈署方式分別是 安裝 及 免安裝

提供 log 掃描的log 每次進行 單次更新時的攝影機狀態 ntp server 的狀態 及每次連入的ip 討論是要寫入 windows 的 log 或是 用txt檔紀錄就好 txt檔紀錄注意檔案大小 數量控制 避免撐爆使用者 電腦

console gui 在每個畫面有可以提示鍵盤操作提示
